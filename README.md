# LabServer: Kubernetes Infrastructure and Monitoring

**Данный проект представляет собой реализацию отказоустойчивой инфраструктуры для образовательной платформы LabServer. Работа выполнена в рамках учебного курса по технологиям разработки современного программного обеспечения. Основной упор сделан на контейнеризацию компонентов, их оркестрацию в среде Kubernetes и настройку автоматизированного мониторинга состояния сервисов.**

## Обзор системы

**Проект LabServer состоит из следующих компонентов:**

* **Backend**: REST API сервер на базе ASP.NET Core 9.0.
* **Frontend Dashboard**: Клиентское приложение для студентов (Angular).
* **Frontend Admin**: Панель управления для администраторов (Angular).
* **Database**: Система управления базами данных PostgreSQL 15.

## Технологический стек

* **Оркестрация**: Kubernetes (Minikube)
* **Среда контейнеризации**: Docker
* **CI/CD компоненты**: Docker Hub (реестр образов)
* **Мониторинг**: Prometheus, Grafana
* **База данных**: PostgreSQL

## Реализация мониторинга (Тема №17)

**В рамках проекта реализована подсистема мониторинга «живучести» (Health Monitoring) компонентов внутри кластера Kubernetes. Основные особенности:**

* **Liveness Probes**: Для каждого критического узла (Backend, Frontend) настроены проверки состояния. Использование **tcpSocket** **позволяет отслеживать готовность сервисов на сетевом уровне.**
* **Self-Healing**: Настроены политики перезапуска контейнеров. В случае отказа TCP-соединения или зависания процесса, Kubernetes автоматически перезапускает под, минимизируя время простоя системы.
* **Параметризация запусков**: Настроены задержки опроса (**initialDelaySeconds**), учитывающие время инициализации тяжелых фреймворков (Angular) и время миграции базы данных.

## Структура репозитория

## Инструкция по развертыванию

### Предварительные требования

* **Установленный Docker Desktop с поддержкой WSL2.**
* **Установленный Minikube.**
* **Утилита управления kubectl.**

### Порядок запуска

* **Склонируйте репозиторий:**

  **code**Bash

  ```
  git clone https://github.com/TriAlexTime/devops-lab-17.git
  cd devops-lab-17
  ```
* **Запустите локальный кластер Kubernetes:**

  **code**Bash

  ```
  minikube start --driver=docker
  ```
* **Разверните инфраструктуру из манифестов:**

  **code**Bash

  ```
  kubectl apply -f k8s/
  ```
* **Дождитесь инициализации всех подов:**

  **code**Bash

  ```
  kubectl get pods --watch
  ```
* **Получите доступ к сервису Dashboard:**

  **code**Bash

  ```
  minikube service dashboard-service
  ```
